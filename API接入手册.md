# 苏适酒店 agent 数字员工 — API 接入手册

> 本手册回答一个问题：**每个 agent 要真正跑在两店 124 间房上，需要接入哪些外部 API？**
>
> 按 agent 拆解 → 每项列候选厂商 / 文档入口 / 申请周期 / 工程量 / 合规提示。

---

## 1. 三阶段落地路径

| 阶段 | 周期 | 范围 | 目标 |
|------|------|------|------|
| **MVP** | 1-2 月 | 数字前台 · 在线客服 · OTA 运营师 | 把订单、客户、渠道打通，立刻见收益 |
| **可运营** | 3-4 月 | 客房管家 · 收益管理师 · 工程运维 | 体验和能耗降本，IoT 开始进场 |
| **深度智能** | 5-6 月 | 宾客画像 · 数字店长 · 数字营销 · 财务风控 · 供应链 · 数字质检 | 全局优化、跨店对标、合规稽核 |

> **强依赖关系**：PMS/CRS 必须第一步解决；OTA 三大平台申请周期长，要 D-Day 前 6 周启动；支付商户号 2 周；企微应用当天能搞定。

## 2. 共享基础设施（所有 agent 共用）

### 2.1 大模型（统一走阿里百炼）

| 模型 | 用途 | API 端点 | 申请周期 |
|------|------|---------|---------|
| **Qwen-Max** | 对话、推理、决策 | `dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | < 1 天 |
| **Qwen-VL-Max** | 证件 OCR + 人脸 | 同上（多模态接口） | < 1 天 |
| **Qwen-Audio** | 语音识别（含方言） | 同上 | < 1 天 |
| **CosyVoice** | 语音合成 | 同上 | < 1 天 |
| **Qwen-Embedding-v3** | 向量嵌入 | 同上 | < 1 天 |

- 注册：阿里云账号 → 百炼控制台 → 创建 API Key
- 文档：<https://help.aliyun.com/zh/dashscope>
- 计费：按 token，无最低消费

### 2.2 PMS / CRS（酒店业务底座 — 最重要！）

所有 agent 共用的"血液层"。**必须先选定一家 PMS 并打通 API。**

| 候选 | 类型 | 国内份额 | API 形式 | 周期 | 备注 |
|------|------|---------|---------|------|------|
| **西软 Fidelio / Opera** | 本地化部署 | 高端酒店多 | REST + Webhook | 2-4 周 | 老牌，接口最全 |
| **金天鹅 Golden Tree** | 本地 / 云 | 国内中端市场大 | REST | 2-3 周 | 生态成熟 |
| **别样红云 PMS** | 纯云 | 中小酒店首选 | REST | 1-2 周 | API 友好、上线快 |
| **住哲酒店管家** | 纯云 | 精品/单体多 | REST | 1-2 周 | 开放度高 |
| **中软卓越 PMSPro** | 本地 | 老牌 | SOAP + REST | 3-4 周 | 接入稍重 |

**必须对接的接口**：
- 订单 CRUD + Webhook（新订单/取消/修改推送）
- 房态查询（房号、类型、状态、清洁度）
- 客人档案（姓名、手机、身份证、会员等级）
- 费用明细（房费、消费、押金）
- 夜审数据

### 2.3 消息中枢

- **阿里云 MQ for RocketMQ**（推荐，生态最全）
- Kafka（自建，如果团队有运维能力）
- Webhook 直连（简单场景）

### 2.4 存储层

| 类型 | 产品 | 用途 |
|------|------|------|
| 关系型 | 阿里云 RDS PostgreSQL | 业务主库 |
| 缓存 | 阿里云 Redis 企业版 | 会话/画像缓存 |
| 向量 | Qdrant / 阿里云 OpenSearch 向量版 | 知识库检索 |
| 时序 | InfluxDB / 阿里云 TSDB | IoT 传感器数据 |
| 数据仓 | 阿里云 ClickHouse | 收益/运营分析 |
| 日志 | 阿里云 Elasticsearch | 审计 + 排障 |
| 对象 | 阿里云 OSS | 证件图片 / 模型制品 |

### 2.5 身份与通信

| 用途 | 产品 | 接入 |
|------|------|------|
| 员工身份 + SSO | 企业微信开放平台 | OAuth |
| 客户 1v1 | 企微客户联系 API | 应用级授权 |
| 员工通知/机器人 | 企微 Webhook | 即用 |
| 外部 SMS | 阿里云 dysmsapi | API Key + 签名报备 |

- 企微文档：<https://developer.work.weixin.qq.com>

## 5. 合规与风险清单

### 数据合规
- **PII 加密**：身份证号、手机号、人脸图像必须加密存储（SM4 或 AES-256）
- **公安联网核查**：入住信息 30 分钟内上报公安治安平台（接入省级公安旅业系统）
- **等保 2.0**：全系统按三级等保建设
- **数据出境**：不允许 PII 流向境外云/模型

### 业务合规
- ⛔ **不爬 OTA C 端**（携程/美团 App）— 违反 TOS，IP 封禁风险
- ✍️ **点评 AI 回复人工确认** — 防止话术不当
- 💰 **支付预授权必须二次人工审核** — 防钓鱼/错扣
- 🔒 **门禁开锁禁止给客服类 Agent** — 权限最小化

### 接入质量
- 所有外部 API 调用必须有**熔断 + 重试 + 超时**（建议用 Sentinel）
- OTA 接口速率限制严格（1-5 QPS），必须**令牌桶**
- 重要业务（支付、开门）走**同步 + 幂等**，不走 MQ 异步

## 6. 推荐开工顺序（Week by Week）

**Week 1-2**：
- 选定 PMS 供应商，签合同
- 申请阿里百炼 + 阿里云基础资源（RDS/Redis/OSS/MQ）
- 启动携程 / 美团 / 抖音商家开发者申请（长周期要尽早）
- 启动支付宝 / 微信支付商户认证

**Week 3-4**：
- PMS 对接完成，基础订单/房态流通
- OCR / 人脸 API 接入
- 数字前台 Agent 先跑通（最纯粹业务，最能见效）

**Week 5-8**：
- OTA 三大平台陆续对接
